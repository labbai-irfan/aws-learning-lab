# Module 7 — 100 VPC Multiple-Choice Questions

Exam-style MCQs covering all 13 topics. Each has 4 options; the **answer + one-line explanation** follows immediately. Cover the answer line and test yourself.

> Format: **Q. question** → A/B/C/D → **✅ Answer — explanation**

---

## Block 1 — VPC & CIDR (Q1–10)

**Q1.** A VPC is scoped to:
A) A single AZ  B) A single Region  C) Globally  D) A single subnet
**✅ B** — A VPC spans all AZs in one Region but never crosses Regions.

**Q2.** Which is a valid VPC CIDR size range?
A) /8 to /16  B) /16 to /28  C) /0 to /32  D) /24 to /32
**✅ B** — AWS allows VPC CIDRs from /16 (largest) to /28 (smallest).

**Q3.** How many usable host IPs in a /24 subnet on AWS?
A) 256  B) 254  C) 251  D) 250
**✅ C** — AWS reserves 5 IPs per subnet, so 256 − 5 = 251.

**Q4.** Which range is NOT RFC 1918 private?
A) 10.0.0.0/8  B) 172.16.0.0/12  C) 192.168.0.0/16  D) 11.0.0.0/8
**✅ D** — 11.0.0.0/8 is public address space.

**Q5.** The default VPC CIDR is:
A) 10.0.0.0/16  B) 172.31.0.0/16  C) 192.168.0.0/16  D) 172.16.0.0/12
**✅ B** — Default VPCs use 172.31.0.0/16.

**Q6.** How many VPCs per Region by default?
A) 1  B) 5  C) 20  D) Unlimited
**✅ B** — 5 by default; raise via service quotas.

**Q7.** Each +1 to a CIDR prefix length:
A) Doubles the range  B) Halves the range  C) No change  D) Quadruples it
**✅ B** — /17 has half the addresses of /16.

**Q8.** To expand a VPC that's out of IPs you:
A) Edit the primary CIDR  B) Add a secondary CIDR block  C) Recreate the VPC  D) Delete a subnet
**✅ B** — Secondary CIDR blocks extend a VPC.

**Q9.** `enableDnsHostnames=true` allows:
A) Faster routing  B) Public DNS names for instances  C) NAT  D) IPv6
**✅ B** — It enables resolvable public DNS hostnames.

**Q10.** For outbound-only IPv6 you use:
A) NAT Gateway  B) Internet Gateway  C) Egress-only internet gateway  D) NAT instance
**✅ C** — NAT is IPv4-only; egress-only IGW is the IPv6 equivalent.

## Block 2 — Subnets (Q11–20)

**Q11.** A subnet lives in:
A) Every AZ  B) Exactly one AZ  C) One Region only  D) Two AZs
**✅ B** — One subnet maps to exactly one AZ.

**Q12.** What makes a subnet "public"?
A) A flag on the subnet  B) Being in AZ-a  C) Route 0.0.0.0/0 → IGW  D) Having IPv6
**✅ C** — A public subnet's route table points internet traffic at the IGW.

**Q13.** Smallest subnet AWS permits:
A) /24  B) /26  C) /28  D) /30
**✅ C** — /28 = 16 IPs (11 usable).

**Q14.** Which 5 IPs does AWS reserve? (count)
A) 2  B) 3  C) 5  D) 0
**✅ C** — Network, router, DNS, future-use, broadcast.

**Q15.** To get HA you spread resources across:
A) Subnets in multiple AZs  B) Multiple route tables  C) Multiple IGWs  D) Multiple NACLs
**✅ A** — Multi-AZ subnets survive AZ failure.

**Q16.** Auto-assign public IP is a property of:
A) The VPC  B) The subnet  C) The route table  D) The NACL
**✅ B** — `mapPublicIpOnLaunch` is a subnet attribute.

**Q17.** Databases should live in:
A) Public subnet  B) Private/isolated subnet  C) The IGW  D) A NAT subnet
**✅ B** — No internet path = smaller attack surface.

**Q18.** Can you change a subnet's AZ?
A) Yes, anytime  B) Only when empty  C) No, recreate it  D) Only via CLI
**✅ C** — AZ is fixed at creation.

**Q19.** A subnet associates with how many route tables?
A) Exactly one  B) Up to five  C) Unlimited  D) Zero
**✅ A** — One route table per subnet (the main one by default).

**Q20.** Two private subnets in the same VPC can communicate because of:
A) NAT  B) The IGW  C) The `local` route  D) Peering
**✅ C** — The non-deletable `local` route connects all subnets.

## Block 3 — Route Tables (Q21–30)

**Q21.** When multiple routes match, AWS picks:
A) The first listed  B) Longest-prefix match  C) Lowest cost  D) Random
**✅ B** — Most specific (longest prefix) wins.

**Q22.** The `local` route can be:
A) Deleted  B) Edited  C) Neither deleted nor edited  D) Disabled
**✅ C** — It's permanent for the VPC CIDR.

**Q23.** A route with a deleted NAT target shows as:
A) Active  B) Pending  C) Blackhole  D) Local
**✅ C** — Invalid-target routes become blackholes; traffic drops.

**Q24.** To make a subnet public, add:
A) 0.0.0.0/0 → NAT  B) 0.0.0.0/0 → IGW  C) local → IGW  D) 10.0.0.0/16 → IGW
**✅ B** — Default route to the internet gateway.

**Q25.** Default route table for an unassociated subnet:
A) Custom RT  B) Main route table  C) None  D) The IGW's RT
**✅ B** — The VPC's main route table.

**Q26.** Which target can a route NOT use?
A) NAT gateway  B) Peering connection  C) Security group  D) Transit gateway
**✅ C** — SGs aren't routing targets; they're firewalls.

**Q27.** For private-subnet internet egress, the route is:
A) 0.0.0.0/0 → IGW  B) 0.0.0.0/0 → NAT GW  C) local → NAT  D) 0.0.0.0/0 → SG
**✅ B** — Private subnets egress via NAT.

**Q28.** A route table can serve:
A) One subnet only  B) Many subnets  C) One AZ  D) The whole Region
**✅ B** — One RT → many subnets.

**Q29.** For 10.0.5.9, which wins: 10.0.0.0/16→A or 10.0.5.0/24→B?
A) A  B) B  C) Tie  D) Neither
**✅ B** — /24 is more specific than /16.

**Q30.** An S3 gateway endpoint adds a route to the table targeting:
A) An IGW  B) A NAT  C) A prefix list → endpoint  D) A peering
**✅ C** — Managed prefix list routed to the VPC endpoint.

## Block 4 — Internet Gateway & Public IPs (Q31–40)

**Q31.** How many IGWs can attach to one VPC?
A) 1  B) 2  C) One per AZ  D) Unlimited
**✅ A** — Exactly one IGW per VPC.

**Q32.** Which is NOT required for internet reachability?
A) IGW attached  B) Route to IGW  C) Public/Elastic IP  D) A NAT gateway
**✅ D** — NAT is for private egress, not public reachability.

**Q33.** The IGW is:
A) A single EC2 you manage  B) AWS-managed, redundant, scalable  C) Bandwidth-limited  D) Per-subnet
**✅ B** — Fully managed and horizontally scaled.

**Q34.** An Elastic IP that is unattached:
A) Is free  B) Incurs charges  C) Auto-releases  D) Can't exist
**✅ B** — Idle EIPs are billed.

**Q35.** A public IP is needed on the instance because:
A) The IGW only translates instances that have one  B) NAT needs it  C) DNS needs it  D) It isn't needed
**✅ A** — Without a public IP the IGW can't NAT it.

**Q36.** Cost of an IGW itself:
A) Hourly  B) Per-GB only (data)  C) Free, pay data transfer  D) Per attachment
**✅ C** — The IGW is free; you pay for data through it.

**Q37.** A private instance routed to the IGW (no public IP) will:
A) Reach the internet  B) Blackhole outbound  C) Get a public IP  D) Use NAT automatically
**✅ B** — No translation possible; packets drop.

**Q38.** Elastic IP remapping lets you:
A) Change Regions  B) Move a static IP between resources  C) Avoid charges  D) Create subnets
**✅ B** — EIPs can be reassigned for failover.

**Q39.** Public subnet instances reach the internet via:
A) NAT  B) The IGW directly  C) Peering  D) Endpoints
**✅ B** — With a public IP + route to IGW.

**Q40.** Which gives an instance a permanent public IP?
A) Auto-assign public IP  B) Elastic IP  C) Private IP  D) DNS hostname
**✅ B** — Auto-assigned IPs change on stop/start; EIPs are static.

## Block 5 — NAT (Q41–50)

**Q41.** A NAT Gateway allows:
A) Inbound from internet  B) Outbound-only from private subnets  C) Both equally  D) Peering
**✅ B** — Outbound-initiated only; inbound is blocked.

**Q42.** A NAT Gateway must be placed in a:
A) Private subnet  B) Public subnet  C) Any subnet  D) Isolated subnet
**✅ B** — It needs a route to the IGW.

**Q43.** NAT Gateway vs NAT instance — the gateway is:
A) Cheaper always  B) Self-managed  C) AWS-managed & auto-scaling  D) A security group
**✅ C** — Managed, HA-in-AZ, up to 100 Gbps.

**Q44.** For NAT high availability you should:
A) Use one shared NAT  B) One NAT per AZ  C) Two NATs in one AZ  D) No NAT
**✅ B** — Per-AZ NAT avoids a SPOF and cross-AZ charges.

**Q45.** Does a NAT Gateway have a security group?
A) Yes  B) No (uses the subnet NACL)  C) Two  D) Optional
**✅ B** — NAT GWs are governed by NACLs, not SGs.

**Q46.** NAT Gateway billing includes:
A) Per-hour only  B) Per-GB only  C) Per-hour + per-GB processed  D) Free
**✅ C** — Both hourly and data-processing charges.

**Q47.** Why can't the internet start a connection through NAT?
A) Firewall blocks it  B) Source NAT only tracks outbound sessions  C) No public IP  D) DNS issue
**✅ B** — Only matching replies to outbound sessions return.

**Q48.** Best way to cut NAT data cost for S3:
A) Bigger NAT  B) S3 gateway endpoint (free)  C) More AZs  D) Disable NAT
**✅ B** — Gateway endpoints remove S3 traffic from NAT.

**Q49.** A NAT Gateway needs:
A) A private IP only  B) An Elastic IP (public NAT)  C) A security group  D) An IGW route inside it
**✅ B** — Public NAT uses an EIP as its source.

**Q50.** A NAT Gateway spans:
A) All AZs  B) One AZ  C) One Region  D) Two AZs
**✅ B** — Single-AZ; deploy one per AZ for resilience.

## Block 6 — NACL vs Security Groups (Q51–65)

**Q51.** Security groups are:
A) Stateless  B) Stateful  C) Subnet-level  D) Deny-capable
**✅ B** — Stateful: returns auto-allowed.

**Q52.** NACLs are:
A) Stateful  B) Instance-level  C) Stateless, subnet-level  D) Allow-only
**✅ C** — Stateless and applied per subnet.

**Q53.** Which can have explicit DENY rules?
A) Security group  B) NACL  C) Both  D) Neither
**✅ B** — Only NACLs support deny.

**Q54.** SG rules are evaluated:
A) By number, first match  B) All together (any allow permits)  C) Top-down with deny  D) Randomly
**✅ B** — SGs are a union of allow rules.

**Q55.** NACL rules are evaluated:
A) All together  B) By rule number ascending, first match  C) By specificity  D) Alphabetically
**✅ B** — Lowest number first, then implicit deny.

**Q56.** SG referencing means:
A) Allowing another SG's ID as source  B) Copying rules  C) Merging SGs  D) Tagging
**✅ A** — Reference SG-id so rules survive scaling.

**Q57.** Default custom security group inbound:
A) Allow all  B) Deny all  C) Allow 22  D) Allow 443
**✅ B** — Denies inbound, allows all outbound by default.

**Q58.** Default (not custom) NACL:
A) Denies all  B) Allows all  C) Allows 443 only  D) Blocks ephemeral
**✅ B** — The default NACL allows all traffic.

**Q59.** A stateless NACL needs an extra rule for:
A) The same port inbound  B) Ephemeral return ports 1024-65535  C) ICMP only  D) Nothing
**✅ B** — Return traffic uses ephemeral ports.

**Q60.** To block one malicious IP for a whole subnet:
A) SG deny  B) NACL deny rule  C) Route blackhole only  D) Impossible
**✅ B** — SGs can't deny; NACL can.

**Q61.** SG allows but NACL denies → result:
A) Allowed  B) Denied  C) Depends on order  D) Logged only
**✅ B** — Both layers must allow.

**Q62.** Security groups attach to:
A) Subnets  B) ENIs/instances  C) Route tables  D) VPCs
**✅ B** — They protect network interfaces.

**Q63.** Max security groups per ENI (default):
A) 1  B) 5  C) 16  D) Unlimited
**✅ B** — 5 by default (raisable).

**Q64.** `ping` between instances fails but HTTP works. Likely:
A) Route missing  B) ICMP not allowed in SG/NACL  C) NAT down  D) DNS
**✅ B** — ICMP must be allowed separately from TCP.

**Q65.** The DB SG should allow 3306 from:
A) 0.0.0.0/0  B) The app SG only  C) The VPC CIDR  D) The IGW
**✅ B** — Least privilege via SG chaining.

## Block 7 — Peering & Transit Gateway (Q66–78)

**Q66.** VPC peering is:
A) Transitive  B) Non-transitive  C) Many-to-many  D) Internet-based
**✅ B** — A↔B and B↔C does not give A↔C.

**Q67.** Peering requires CIDRs that:
A) Overlap  B) Do not overlap  C) Are identical  D) Are public
**✅ B** — Overlapping ranges can't be routed.

**Q68.** After creating a peering you must:
A) Nothing  B) Add routes on both VPCs  C) Add an IGW  D) Add a NAT
**✅ B** — Routes (and SG/NACL allows) on both sides.

**Q69.** Peering traffic travels over:
A) The public internet  B) The AWS backbone (private)  C) A VPN  D) NAT
**✅ B** — Private, never the internet.

**Q70.** Peering can be:
A) Same-Region only  B) Same-account only  C) Cross-Region & cross-account  D) IPv6 only
**✅ C** — Both cross-Region and cross-account supported.

**Q71.** For many VPCs, peering becomes a problem because:
A) It's transitive  B) Full-mesh scaling (N(N-1)/2)  C) It's too cheap  D) It needs NAT
**✅ B** — Mesh explosion; use TGW instead.

**Q72.** Transit Gateway topology:
A) Mesh  B) Hub-and-spoke  C) Ring  D) Star of IGWs
**✅ B** — Central hub router.

**Q73.** TGW supports transitive routing:
A) No  B) Yes  C) Only same-AZ  D) Only IPv6
**✅ B** — Spokes can reach each other via the hub.

**Q74.** TGW segmentation (e.g. isolate Dev from Prod) uses:
A) NACLs  B) TGW route tables  C) Security groups  D) IGWs
**✅ B** — Route-table associations control reachability.

**Q75.** TGW billing includes:
A) Free  B) Per-attachment-hour + per-GB  C) Per-subnet  D) Per-route
**✅ B** — Attachment hours plus data processing.

**Q76.** To share a TGW across accounts use:
A) IAM only  B) Resource Access Manager (RAM)  C) Peering  D) S3
**✅ B** — RAM shares the TGW in an Organization.

**Q77.** Connect TGWs in different Regions via:
A) Peering attachment  B) NAT  C) IGW  D) VPN only
**✅ A** — Inter-Region TGW peering.

**Q78.** On-prem connects to a TGW via:
A) IGW  B) VPN / Direct Connect  C) Peering  D) Endpoint
**✅ B** — Site-to-Site VPN or Direct Connect.

## Block 8 — VPC Endpoints (Q79–88)

**Q79.** Which services use GATEWAY endpoints?
A) All services  B) S3 and DynamoDB only  C) SSM only  D) RDS only
**✅ B** — Gateway endpoints serve S3 and DynamoDB.

**Q80.** Gateway endpoints cost:
A) Per-hour  B) Per-GB  C) Free  D) Per-AZ
**✅ C** — Gateway endpoints are free.

**Q81.** Interface endpoints are based on:
A) Route tables  B) An ENI/private IP (PrivateLink)  C) NAT  D) IGW
**✅ B** — A private ENI controlled by an SG.

**Q82.** Interface endpoint billing:
A) Free  B) Per-AZ-hour + per-GB  C) Per-route  D) Per-subnet
**✅ B** — Hourly per-AZ plus data.

**Q83.** A gateway endpoint is added to the VPC via:
A) An ENI  B) A route-table prefix-list entry  C) A NAT  D) A peering
**✅ B** — Managed prefix list in the route table.

**Q84.** Endpoints keep AWS-service traffic:
A) On the internet  B) Private on the AWS backbone  C) Through NAT always  D) Encrypted only
**✅ B** — No internet/IGW/NAT path.

**Q85.** To restrict what an endpoint can access use:
A) Route priority  B) An endpoint policy  C) A NAT rule  D) DNS
**✅ B** — Endpoint (resource) policies.

**Q86.** "Private DNS" on an interface endpoint:
A) Speeds DNS  B) Resolves the public service name to the private IP  C) Adds IPv6  D) Disables DNS
**✅ B** — So apps need no code change.

**Q87.** Best replacement for NAT when only S3 is needed:
A) Bigger NAT  B) S3 gateway endpoint  C) IGW  D) Peering
**✅ B** — Free, private, removes NAT data cost.

**Q88.** PrivateLink lets you expose:
A) Only AWS services  B) Your own service to other VPCs/accounts  C) Public IPs  D) NAT
**✅ B** — Private service sharing via interface endpoints.

## Block 9 — Packet/Request/Internet Flow (Q89–95)

**Q89.** Outbound packet order (simplified) is:
A) NACL → SG → Route  B) SG → Route → NACL → gateway  C) Route → SG → IGW  D) NACL → Route → SG
**✅ B** — SG (instance) → route table → NACL (subnet) → gateway.

**Q90.** A reply succeeds with SGs but fails with a custom NACL. Cause:
A) Route missing  B) NACL missing ephemeral return rule  C) NAT down  D) DNS
**✅ B** — Stateless NACLs need explicit return rules.

**Q91.** In SG chaining, the app tier allows traffic from:
A) 0.0.0.0/0  B) The ALB's SG  C) The DB SG  D) The IGW
**✅ B** — App trusts only the load balancer SG.

**Q92.** To be reachable FROM the internet you need:
A) NAT  B) Public subnet + IGW + public IP  C) Endpoint  D) Peering
**✅ B** — The public reachability triad.

**Q93.** To reach OUT only (private) you need:
A) IGW + public IP  B) NAT Gateway  C) Peering  D) Endpoint policy
**✅ B** — NAT for outbound-only egress.

**Q94.** Best path for a private instance to reach S3:
A) NAT → IGW  B) VPC (gateway) endpoint  C) Peering  D) Public IP
**✅ B** — Private, free, no internet.

**Q95.** The `local` route guarantees:
A) Internet access  B) Intra-VPC connectivity (subject to SG/NACL)  C) NAT  D) DNS
**✅ B** — All subnets can route to each other.

## Block 10 — Architecture & Troubleshooting (Q96–100)

**Q96.** In a 3-tier design, the data tier subnet route table has:
A) 0.0.0.0/0 → IGW  B) 0.0.0.0/0 → NAT  C) local only  D) 0.0.0.0/0 → endpoint
**✅ C** — Isolated: no internet route at all.

**Q97.** ALB targets show unhealthy. Most likely:
A) Wrong Region  B) Target SG doesn't allow ALB SG / wrong health path  C) No DNS  D) IGW missing
**✅ B** — SG/health-check misconfig is the usual cause.

**Q98.** Surprise data-transfer bills usually come from:
A) Subnets  B) NAT data + cross-AZ transfer  C) NACLs  D) Route tables
**✅ B** — These dominate the VPC bill ("EC2-Other").

**Q99.** Best tool to answer "why can't A reach B?":
A) CloudTrail  B) VPC Reachability Analyzer  C) Cost Explorer  D) Trusted Advisor
**✅ B** — It evaluates the full path automatically.

**Q100.** For many VPCs + on-prem with environment isolation, choose:
A) Peering mesh  B) Transit Gateway with route tables  C) One big VPC  D) NAT instances
**✅ B** — Hub-and-spoke with segmentation scales correctly.
