# Module 9 — 50 S3 Scenario Questions (with Solutions)

> Real "what would you design/do" situations. Decide first, then check the **Solution**. Mirrors Solutions Architect interviews and on-call decisions.

---

### Storage Classes & Cost
**1.** Logs are hot for 30 days, rarely read after, must be kept 1 year.
→ **Lifecycle:** Standard → Standard-IA (30d) → Glacier (90d) → expire (365d). Cheap and automatic.

**2.** Access pattern for a dataset is unpredictable.
→ **S3 Intelligent-Tiering** — auto-tiers with no retrieval fees; set-and-forget.

**3.** Compliance requires keeping records 7 years, rarely accessed.
→ **Glacier Deep Archive** + lifecycle + Object Lock for retention.

**4.** Thumbnails can be regenerated from originals anytime.
→ **One Zone-IA** for the thumbnails (cheaper; loss is recoverable by regenerating).

**5.** Storage bill keeps growing despite stable data volume.
→ Likely **versioning bloat / incomplete multipart uploads**; add NoncurrentVersionExpiration + AbortIncompleteMultipartUpload; check with Storage Lens.

**6.** A team puts small, frequently-updated files in Standard-IA and costs rose.
→ Move them back to **Standard** — IA's min size/early-delete fees penalize small/changing objects.

**7.** Need to cut data-transfer-out costs for a popular download.
→ Front S3 with **CloudFront** (edge caching reduces origin GETs and egress).

**8.** Want org-wide visibility into S3 waste.
→ **S3 Storage Lens** (+ Inventory) and cost allocation tags.

### Versioning, Backup & DR
**9.** An engineer overwrote a critical file.
→ **Versioning** lets you restore the previous version (enable it proactively!).

**10.** Protect backups from ransomware/insider deletion.
→ **Versioning + Object Lock (WORM) + MFA Delete**, replicate to another Region.

**11.** Need a disaster-recovery copy in another Region.
→ **Cross-Region Replication (CRR)** (requires versioning on both buckets).

**12.** Aggregate logs from several buckets into one in the same Region.
→ **Same-Region Replication (SRR)**.

**13.** You just enabled replication but old objects didn't copy.
→ Replication only handles **new** objects; run **S3 Batch Replication** for existing ones.

**14.** Migrate 500 TB from on-prem with limited bandwidth.
→ **AWS Snowball** (physical device); use DataSync for ongoing sync.

**15.** Backups must be restorable — how do you ensure it?
→ **Regularly test restores**; a backup you can't restore is worthless. Automate with AWS Backup.

**16.** Need point-in-time protection plus cheap long-term retention.
→ Versioning + lifecycle transition of noncurrent versions to Glacier, expire after retention.

### Access & Security
**17.** Marketing accidentally made a bucket public; data leaked.
→ Turn **Block Public Access ON**, audit with **Access Analyzer**, rotate exposed data; share via pre-signed/CloudFront instead.

**18.** Let users download private files without AWS credentials.
→ **Pre-signed GET URLs** (short expiry) or **CloudFront signed URLs**.

**19.** Let users upload directly to S3 without exposing the bucket.
→ **Pre-signed PUT URLs** generated by your server; bucket stays private.

**20.** Another AWS account needs read access to a bucket.
→ **Bucket policy** (or Access Point) granting that account's role, plus their IAM allow.

**21.** Enforce that all access uses HTTPS.
→ Bucket policy **Deny** when `aws:SecureTransport=false`.

**22.** Enforce that all uploads are KMS-encrypted.
→ Bucket policy denies PutObject without `x-amz-server-side-encryption: aws:kms`.

**23.** Cross-account uploads are owned by the uploader, not you.
→ Set **Object Ownership = Bucket owner enforced** (disables ACLs) or require `bucket-owner-full-control`.

**24.** App role currently has `s3:*` on `*`.
→ Apply **least privilege**: only GetObject/PutObject on the specific prefix.

**25.** Need to know who downloaded a sensitive object.
→ Enable **CloudTrail data events** + S3 server access logging.

**26.** Find PII accidentally stored in S3.
→ **Amazon Macie**.

**27.** A pre-signed upload works in Postman but fails in the browser.
→ **CORS** not configured; add a CORS rule for your origin + PUT method.

**28.** Pre-signed PUT fails with SignatureDoesNotMatch.
→ Browser's **Content-Type must match** what was used to sign; align headers.

**29.** Pre-signed URLs are being shared and reused.
→ Use **short expiry**, per-object scope, and authenticate before issuing.

**30.** Sensitive HR docs need key-level access control + audit.
→ **SSE-KMS** with a dedicated key; restrict decrypt to HR roles; CloudTrail logs decrypts.

### Static Hosting & Delivery
**31.** Host a React SPA cheaply with HTTPS and a custom domain.
→ **S3 (private) + CloudFront + OAC + ACM + Route 53**; error document = index.html for routing.

**32.** Static site on S3 works but has no HTTPS.
→ S3 website endpoint is HTTP-only; **put CloudFront in front** for HTTPS.

**33.** Deployed new files but users see old content.
→ **CloudFront invalidation** for changed paths (or versioned filenames).

**34.** Serve a global audience with low latency.
→ **CloudFront** edge caching (optionally CRR for origin redundancy).

**35.** Protect premium videos from free sharing.
→ **CloudFront signed URLs/cookies** + private origin.

### Architecture Design
**36.** Design a scalable user file-upload feature.
→ React → Node issues **pre-signed PUT** → browser uploads **directly to a private bucket** → DB stores metadata → optional S3 event → Lambda processing → pre-signed GET/CloudFront for reads.

**37.** Build an image platform with thumbnails.
→ Upload originals → **S3 event → Lambda** generates thumbnails to a processed bucket → serve via CloudFront.

**38.** Build a video platform.
→ Originals to raw bucket → **MediaConvert** transcodes renditions → processed bucket → CloudFront (signed URLs for paid content).

**39.** Build a data lake landing zone.
→ S3 buckets by zone (raw/clean/curated), lifecycle tiering, partitioned prefixes, query with Athena, govern with Lake Formation.

**40.** Build an HRMS document store (compliance).
→ Private bucket, SSE-KMS dedicated key, per-employee prefixes + policy conditions, versioning + Object Lock retention, CloudTrail audit, CRR for DR, short-lived pre-signed URLs.

**41.** Reduce server load from large uploads/downloads.
→ **Direct-to-S3 via pre-signed URLs** + CloudFront — the server never proxies bytes.

**42.** Process uploaded files asynchronously and reliably.
→ S3 event → **SQS → Lambda/worker** (decouples + retries + DLQ).

**43.** Allow partners to drop files into specific folders only.
→ **S3 Access Points** or scoped bucket policies per prefix; pre-signed URLs for uploads.

**44.** Keep raw and processed media separate.
→ **Two buckets** with different lifecycle/cost and a clean pipeline between them.

### Performance & Operations
**45.** Uploads from a distant country are slow.
→ **S3 Transfer Acceleration** (or multipart with more parallelism; CloudFront for downloads).

**46.** A 20 GB file upload keeps failing.
→ Use **multipart upload** (single PUT max is 5 GB); add abort-incomplete lifecycle rule.

**47.** Worried about request rate limits on a hot prefix.
→ S3 **scales automatically**; for extreme cases spread keys, but usually no action needed.

**48.** Need a scheduled inventory of all objects for audit.
→ **S3 Inventory** report (CSV/Parquet) to a bucket.

**49.** SSE-KMS is causing throttling at high upload volume.
→ Enable **S3 Bucket Keys** to cut KMS API calls.

**50.** Want to guarantee files can't be deleted for a fixed period.
→ **S3 Object Lock** in compliance mode (WORM) for the retention duration.

## Pattern Reflexes (memorize)
```
"share private file"        → pre-signed URL / CloudFront signed URL
"direct browser upload"     → pre-signed PUT + CORS, private bucket
"unknown access pattern"    → Intelligent-Tiering
"cold/compliance archive"   → Glacier Deep Archive + Object Lock
"DR across Regions"         → CRR (needs versioning)
"global low latency"        → CloudFront + OAC (private origin)
"protect from delete"       → versioning + Object Lock + MFA Delete
"HTTPS static site"         → S3 private + CloudFront + ACM
"sensitive data"            → SSE-KMS + least privilege + CloudTrail
"cut storage cost"          → lifecycle + expire versions + abort MPU
"post-upload processing"    → S3 event → Lambda/SQS
"found public bucket"       → Block Public Access ON + audit
```
